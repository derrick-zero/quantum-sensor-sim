Class SimulationEngine

SimulationEngine orchestrates the simulation by updating sensors and sensor spheres, processing physics interactions, handling collisions, and enforcing container boundaries. It supports starting, pausing, resetting, randomizing, and toggling time reversal.

Private handleCollision

  • handleCollision(sensor1, sensor2, distanceVector, _distance): void

  • Handles collision response between two sensors using an impulse-based elastic collision model.

    Conservation Principles:

    • Momentum Conservation: The impulse is computed so that the total momentum is preserved: m₁·v₁ + m₂·v₂ = m₁·v₁' + m₂·v₂'

    • Kinetic Energy Conservation: For a perfectly elastic collision, kinetic energy is conserved: ½·m₁·v₁² + ½·m₂·v₂² = ½·m₁·v₁'² + ½·m₂·v₂'²

    Performance Considerations: This impulse-based method calculates the post-collision velocities in one step using the impulse formula, which is computationally efficient compared to iterative “brute-force” methods that adjust velocities incrementally until the conservation conditions are met. This one-pass approach is especially beneficial under high collision frequencies.

    Parameters

    • sensor1: Sensor

      The first sensor involved in the collision.

    • sensor2: Sensor

      The second sensor involved in the collision.

    • distanceVector: Vector3

      The vector from sensor1 to sensor2.

    • _distance: number

      The distance between the centers of the two sensors.

    Returns void

    Remarks

    This method assumes perfectly elastic collisions. Any introduction of inelastic factors (energy loss, damping, etc.) would require modifying the impulse calculation accordingly.
